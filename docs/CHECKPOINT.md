# Чекпоинт

## Что проверить перед отправкой

### Код

Запустите:

```bash
anchor build
cd packages/backend && npm run build
cd ../frontend && npm run build
```

Если нужно проверить тесты:

```bash
anchor test
```

## Что проверить в работе продукта

1. Открывается главная страница.
2. Работает вход в систему.
3. Подключается кошелек.
4. Создается vault.
5. Работает пополнение.
6. Отправляется запрос на расход.
7. Появляется решение AI или безопасный запасной режим.
8. История решений читается понятно.

## Что показать на демо

### Короткий путь

1. Войти
2. Подключить кошелек
3. Создать vault
4. Пополнить vault
5. Отправить запрос
6. Показать:
   - оценку AI
   - проверку правил
   - итоговое решение
   - запись в Solana

## Что не должно попасть в git

Проверьте `git status`.

В списке не должно быть:

- `.env`
- локальных баз
- логов
- временных файлов
- ключей
- локальных сборок

### Frontend


```text
http://localhost:5173/
```

## Если что-то сломалось

### Не отвечает backend

Проверьте:

- заполнен ли `packages/backend/.env`
- есть ли `GEMINI_API_KEY`
- хватает ли SOL у backend ключа

### Не открывается frontend

Проверьте:

- заполнен ли `packages/frontend/.env`
- не занят ли порт `5173`

### Не создается vault

Проверьте:

- подключен ли кошелек
- хватает ли SOL
- совпадает ли `risk authority`

## Что главное сказать на защите

Aegis не дает деньгам уходить сразу.
Система сначала проверяет запрос, потом применяет правила и только после этого принимает решение в сети.
